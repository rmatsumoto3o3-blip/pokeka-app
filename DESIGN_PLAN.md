# PokéLix リニューアル デザイン構成計画書

作成日: 2026-07-07
対象: PokéLix トップページ / 共通シェル / 攻略トップUI

---

## 1. 前提

PokéLix はもともと「Pocket analytics」の略語であり、ポケットの中で持ち運べる分析サイトという意味を持つ。

一方で、最初に攻略サイトとして認知された入口はポケカの環境考察である。既存会員・既存訪問者にとっては「PokéLix = ポケカ攻略・環境分析・便利ツール」という認知が強い。

そのため、リニューアルでは親ブランドを別名に置き換えるのではなく、PokéLix の認知を維持したまま、他TCGの環境考察へ拡張する構成にする。

## 2. コンセプト

### 主軸
ポケカ攻略を中心にした、持ち運べるTCG環境分析サイト。

### 表現方針
- 「総合TCGサイトに生まれ変わった」ではなく、「ポケカ攻略のPokéLixが他TCGにも広がる」と見せる。
- 既存ユーザーが迷わないよう、ファーストビューの主語は PokéLix / ポケカ攻略にする。
- 他TCG対応は、ゲームタブと右カラムで「順次対応」として自然に見せる。

## 3. ブランド表示

ヘッダー左上は `PokéLix` を維持する。
補助ラベルとして `Pocket analytics` を小さく添える。これにより、ブランド名の由来を残しつつ、単なるポケカ専用名ではなく「分析サイト」であることを補強する。

`TCGLIX` は現時点では前面に出さない。必要であれば内部プロジェクト名、または将来の上位カテゴリ名として扱う。

## 4. ファーストビュー構成

### 目的
初見ユーザーに以下をすぐ伝える。
- ここはポケカ攻略・環境分析サイトである。
- 環境データと練習ツールがある。
- 他TCGの環境考察も順次対応する。

### 構成
1. ブランドロゴ
2. 検索ボックス
3. グローバルナビ
4. ゲームタブ
5. ヒーロー
6. 主要入口カード
7. 広告枠

### ヒーロー文言
見出し:
```
ポケカの環境考察と練習ツールを、
ポケットの中に。
```

本文:
```
PokéLix は大会データをもとに、環境Tier・採用率・一人回し・初手確率をまとめて確認できる攻略サイトです。
ポケカを中心に、他TCGの環境考察も同じ見やすさで広げていきます。
```

改行はブラウザ任せにせず、意味単位で制御する。

## 5. 主要入口

ファーストビュー内に、最重要導線として3つの入口を置く。

1. **ポケカ環境Tier表** — 大会データから今強いデッキを見る入口。
2. **一人回し練習** — 既存PokéLixの主力ツールとして目立たせる。
3. **初手確率シミュレーター** — 確率検証ツールとして一人回しと並べる。

この3つを最初に見せることで、「攻略メディア」と「ツールサイト」の両方の価値を同時に伝える。

## 6. ゲームタブ

ヘッダー下にゲームタブを配置する。

初期状態:
- ポケカ: 環境考察・ツール公開中
- ワンピ: 環境考察 準備中
- 遊戯王: 準備中
- デュエマ: 準備中

ゲームタブは将来の多TCG対応を示すが、現時点ではポケカが主役であることを明確にする。

## 7. メインコンテンツ

### 今週のポケカ環境
Tier表を看板コンテンツとして配置する。

表示要素:
- Tierランク
- デッキ名
- 更新日
- 優勝シェア
- 前週比
- 傾向矢印

傾向矢印は以下の3種で統一する。
```
↗ 上昇
↘ 下降
→ 横ばい
```
矢印はサイズ差が出ないよう、専用の固定サイズボックスと欧文フォントで表示する。

### 無料ツール
既存PokéLixらしさを残すため、画像サムネイル付きカードで配置する。

対象:
- 一人回し練習
- 初手確率シミュレーター
- サイド推論トレーナー
- 環境デッキ分布

ツールは「すぐ触れる」印象を優先し、説明文は短くする。

### 環境考察・記事
大会まとめ、環境分析、構築論を一覧化する。

表示要素:
- 記事カテゴリ
- 記事タイトル
- NEW / 更新 / 分析 バッジ
- 日付

## 8. サイドカラム

PC幅では右カラムを置く。

### 他TCGも順次対応
PokéLixの見やすさを他TCGにも広げる構想を表示する。

対象:
- ワンピースカード
- 遊戯王OCG
- デュエマ

ここでは「準備中」であることを明示し、ポケカ中心の認知を壊さない。

### 人気ランキング
訪問者が迷わず主要コンテンツへ移動できるようにする。

例:
- 環境Tier表
- 一人回し練習
- 初手確率シミュレーター
- 採用率ランキング
- 入賞デッキまとめ

## 9. 広告配置

既存サイトで設定している広告・スポンサー枠は維持する。
ヒーロー直下に広告帯を配置する。

内容:
- TOYGER広告
- AdSense
  - slot: `1093986865`
- ドット絵提供者様リンク枠

PCでは横3列、スマホでは縦並びにする。
広告は収益上必要な要素なので、後から無理に差し込むのではなく、初期レイアウトから余白と視線の流れに含める。

## 10. デザインルール

### 色
基調は白。
看板色は青。

青は以下に限定して使う。
- ヘッダー下のゲームタブ
- ヒーローのアクセントバー
- 見出しの左ライン
- 主要リンク
- ランキング数字
- Tier表の重要指標

### 角丸
基本は `8px`。
丸すぎるカードや、要素ごとにバラついた角丸は避ける。

### 余白
高密度にしすぎず、既存PokéLixの軽さを残す。
ただし、情報の所在が曖昧にならないよう、セクション単位で枠と見出しを明確にする。

### 改行
見出しと重要説明文はブラウザ任せにしない。意味単位で改行する。

悪い例:
```
ポケカの環境考察と練習
ツールを、ポケットの中に。
```
良い例:
```
ポケカの環境考察と練習ツールを、
ポケットの中に。
```

### フォント
日本語攻略サイトとして自然に読めるよう、以下を優先する。
```
Hiragino Sans / Yu Gothic / Meiryo / system-ui
```
海外SaaS寄りに見えるフォント感は避ける。

## 11. SEO・既存導線への配慮

既存SEO資産を壊さないため、既存URLは維持する。
特に以下は変更しない。
- `/practice`
- `/simulator`
- `/archetypes/*`
- `/articles/*`

トップページの見た目と情報設計を刷新しても、主要ツールへの導線は既存URLへ向ける。

## 12. 実装フェーズ案

### Phase 1
共通ヘッダー、ゲームタブ、トップ構成、広告帯を導入する。対象はポケカのみ。

### Phase 2
Tier表と環境デッキ一覧を、実データからメディア型UIへ再構成する。

### Phase 3
他TCGの環境考察を追加する。最初は「準備中」表示から始め、データ収集が成立するTCGから公開する。

## 13. 現時点の判断

このリニューアルは、PokéLixを別ブランドへ置き換えるものではない。
既存会員にとっては「PokéLixが見やすくなった」と感じられることを優先する。
そのうえで、新規訪問者には「ポケカ中心だが、他TCGの環境考察も見られる分析サイト」と伝える。
